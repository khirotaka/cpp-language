# Chapter 4

## 分割コンパイル
コンパイルの直前に、ヘッダファイルの読み込みや、マクロの展開などの前処理が行われる。このことを**プリプロセス** という。  
1つのファイルから1つのオブジェクトファイルが生成され、それらを結合して実行形式ファイルを生成する。この手順を**リンク** という。

### extern変数
ヘッダファイルや複数のソースファイルでグローバル変数を宣言すると、それぞれのソースファイルがコンパイルされた時に、
それぞれのオブジェクトファイルがグローバル変数の実態を持つようになる。

`extern` が付けられて変数は、グローバル変数の実体は何所か別に所で定義されているという意味になる。


## inline関数
関数の規模が小さい一方、関数呼び出しにかかるコストが大きい場合、後者のせいでパフォーマンスが悪化することがある。  
層行った場合、関数の処理内容と同じことを呼び出し元で行った方が早い。それを行う操作が `inline` 展開

これは、関数宣言時に `inline` キーワードを付け加える。


## 名前空間

```cpp
namespace NapespaceName {
    body;
}
```

名前空間はネストすることができる。

```cpp
namespace library {
    namespace module {
        namespace detail {
            void internal_func();
        }
    }
}
```


int main() {
    library::module::detail::internal_func();
}
```

上の書き方以外にも、

```cpp
namespace toplevel::nested {
    ...
}
```


### 無名名前空間
異なるソースファイルに存在する同じ名前を区別するのに使う。  
これを使うことで、実装されているソースファイルの中のみで有効な名前として使うことができる。
